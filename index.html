About dm-Forms

dm-Forms is a tool that was developed using Scala to help create and handle html forms in play in an easier to use way than the way that is defaultly implemented in Play. To use dm-forms with all of its functions you'll need to implement Twitter Bootstrap, jQuery, bootstrap-timepicker, and bootstrap-datepicker which can all be found and implemented with webjars. You do not have to use any or all of these for dm-Forms to work, but some features will not be available without them.

Basic Form Implementation

To create a basic form you'll need to import the following: 

import org.dupontmanual.forms._
import org.dupontmanual.forms.fields._
Once these are imported you can go ahead and create a new form object as follows: 
object BasicFormExample extends Form {
}
After the form object has been created, you can go ahead and create whichever fields you would like (see our api page for more details about available fields) within the form object and add them to the list of fields in the order you wish for them to appear as follows: 
object BasicFormExample extends Form {
    val numericField = new NumericFieldOptional[Double]("Double")
    val passwordField = new PasswordFieldOptional("Password")
    val textField = new TextFieldOptional("Text")
    val fileField = new FileFieldOptional("File")
    val fields = List(numericField, passwordField, textField, fileField)
}
Now that we have created our form, we can go ahead and wrap it in a binding, and then send it to our views. 
def handleGetRequest() = Action { implicit req =>
    Ok(some.view(Binding(BasicFormExample)))
}
Now in the view you need simply to call @Binding.render() wherever you wish the form to display in your html, and it will automatically render itself!

Handling User Input

In order to handle a user's input you'll need to make sure that there were no errors first, which can be done by testing the type of Binding that was returned. If there are errors you probably want to display the errors for the user to fix, and if not, you'll want to grab the values returned (see the api for what values are returned for each field). Example using the basic form object created above: 

def handlePostRequest() = Action { implicit req =>
  Binding(BasicFormExample, req) match {
    
    //To display the form and any errors
    case bindingWithErrors: InvalidBinding => Ok(some.view(bindingWithErrors))
    
    //To get values from the fields
    case goodBinding: ValidBinding => {
      val numericFieldData = goodBinding.valueOf(BasicFormExample.numericField)
      val passwordFieldData = goodBinding.valueOf(BasicFormExample.passwordField)
      val textFieldData = goodBinding.valueOf(BasicFormExample.textField)
      val fileFieldData = goodBinding.valueOf(BasicFormExample.fileField)
      //Do whatever you want with the values.
      Ok(some.view())
    }
  }
}

Dependencies

As mentioned above, dm-Forms does have dependencies on Twitter Bootstrap, jQuery, bootstrap-timepicker, and bootstrap-datepicker. Below each dependency is listed with what is dependent on it.

jQuery

Bootstrap is dependent upon jQuery.

Bootstrap

Bootstrap-timepicker and bootstrap-datepicker are both dependent upon Bootstrap. Along with these, BaseAutocompleteField and its subclasses use Bootstrap's typeahead javascript in their widgets, field help text uses Bootstrap's tooltip javascript, and ValidationErrors use Bootstrap's validation states' base CSS to appear.

Bootstrap-timepicker

BaseTimeField, BaseDateTimeField, and their subclasses all use bootstrap-timepicker in their widgets.

Bootstrap-datepicker

BaseDateField, BaseDateTimeField, and their subclasses all use bootstrap-datepicker in their widgets.
